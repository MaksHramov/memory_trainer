# Требования к ПО — Тренажёр памяти и внимания «Memory Trainer»

Разработка веб-сервиса начинается с процесса формулирования требований, в котором, исходя из потребностей пожилых пользователей, результатов исследования когнитивных тренажёров и опыта разработчика, должен быть создан документ, максимально точно определяющий задачи команды. Этот документ — внешнее описание ПС или требования к ПС.

---

# Методы сбора и извлечения требований

## Анализ предметной области

Программное решение реализует модель предметной области. Для тренажёра памяти предметная область — поддержание когнитивных функций у пожилых людей, проведение простых упражнений на память и внимание, учёт прогресса пользователя.

Анализом предметной области называют деятельность, направленную на:

- выявление реальных потребностей пожилых пользователей (простой интерфейс, крупный шрифт, понятные инструкции)
- выяснение смысла высказанных пожеланий пользователей и их родственников
- выявление свойств желаемых результатов (сохранение истории прохождений)
- определение набора задач для их достижения
- определение набора сущностей, участвующих в решении задач
- определение границ ответственности программной системы

**Источники требований:**

- беседы с пожилыми пользователями и их близкими
- изучение существующих когнитивных тренажёров и приложений для brain training
- рекомендации геронтологов и специалистов по когнитивной реабилитации
- наблюдение за поведением пользователей при работе с цифровыми сервисами
- анализ типовых упражнений на память (запоминание слов) и внимание (поиск отличий)

## Задача 1 — Концепция проекта

### 1. Описание проблемы

| | |
|---|---|
| **Проблема** | С возрастом снижается скорость запоминания и концентрация внимания; пожилым людям сложно пользоваться перегруженными приложениями |
| **Воздействует на** | Людей старшего возраста (60+), а также их родственников и caregivers, заинтересованных в поддержании когнитивного здоровья |
| **Результатом чего является** | Снижение когнитивных функций, отсутствие регулярных тренировок, сложность отслеживания прогресса |
| **Выигрыш от ПП** | Доступные упражнения дома, понятный интерфейс, сохранение результатов, мотивация через наглядный прогресс |

### 2. Назначение программного средства

| Вопрос | Ответ |
|---|---|
| Кто применяет ПС для решения своих задач? | Пожилые пользователи; родственники, помогающие с регистрацией и просмотром результатов |
| От каких систем/пользователей поступают входные данные? | Пользователь вводит логин и пароль; результаты тестов формируются на клиенте и отправляются на сервер |
| Кто/что использует результаты? | Пользователь через вкладку «Результаты»; сервер хранит историю в PostgreSQL |
| Какие сторонние системы участвуют в обработке? | PostgreSQL для хранения данных; Docker для развёртывания |

### 3. Контекстная диаграмма

```
                         ┌─────────────────────────┐
  [Пользователь]  ─────► │                         │
  (веб-браузер)          │     Memory Trainer      │ ──────► [PostgreSQL]
                         │  (Тренажёр памяти)      │
  [Родственник]   ─────► │                         │ ──────► [История результатов]
  (помощь с входом)      └─────────────────────────┘
```

## Задача 3 — Анализ конкурентов

### Сравнительная характеристика конкурирующих решений

| Проблема | Сегмент рынка | Lumosity | Elevate | Простые веб-тренажёры | Приоритет |
|---|---|---|---|---|---|
| Интерфейс, адаптированный для пожилых | 60+ | - | - | +/- | e |
| Тест на запоминание слов | Все | + | + | + | e |
| Тест на внимание | Все | + | + | - | e |
| Сохранение истории результатов | Все | + | + | - | e |
| Регистрация и личный кабинет | Все | + | + | - | u |
| Русскоязычный интерфейс | RU | - | - | +/- | e |
| Бесплатный локальный запуск | Разработка | - | - | +/- | u |
| Мобильное приложение | Все | + | + | - | d |

> **Условные обозначения:** `e` — обязательно (essential), `u` — полезно (useful), `d` — желательно (desirable)

### Вывод по анализу конкурентов

Крупные зарубежные платформы ориентированы на молодую аудиторию и англоязычный интерфейс. **Memory Trainer** решает задачу доступного русскоязычного тренажёра с крупным UI, двумя базовыми упражнениями и сохранением персональной истории результатов — без лишней сложности для пожилого пользователя.

**«для»** — пожилых людей и их близких, заботящихся о когнитивном здоровье;

**«который»** — испытывает трудности с запоминанием и концентрацией внимания;

**«этот»** — **Memory Trainer**;

**«является»** — веб-тренажёром памяти и внимания;

**«который»** — предлагает простые упражнения с крупным шрифтом, сохраняет результаты и показывает историю прохождений;

**«в отличие от»** — сложных мобильных приложений с перегруженным интерфейсом;

**«наш продукт»** — лёгкий веб-сервис на React + Spring Boot с фокусом на доступность для людей старшего возраста.

## Задача 5 — Моделирование сценариев использования

### 1. Иерархия заинтересованных лиц

```
Заинтересованные лица
├── Пожилой пользователь
│   ├── Самостоятельно проходит тесты
│   └── Просматривает свои результаты
├── Родственник / опекун
│   ├── Помогает зарегистрироваться
│   └── Отслеживает прогресс по результатам
└── Система (не человек)
    └── PostgreSQL — хранение пользователей и результатов
```

### 2. Сценарии использования

#### Актор: Пожилой пользователь

**UC-01: Регистрация и вход**

1. Пользователь открывает страницу регистрации
2. Вводит логин, пароль и повтор пароля
3. Система создаёт аккаунт и выполняет автоматический вход
4. Пользователь попадает на страницу тестов

*Альтернативный сценарий:* Пользователь уже зарегистрирован → переходит на страницу входа и авторизуется по логину и паролю

**UC-02: Прохождение теста на память**

1. Пользователь открывает вкладку «Память»
2. Нажимает «Начать тест»
3. Запоминает 5 слов за 25 секунд
4. Выбирает запомненные слова из списка
5. Система показывает результат и сохраняет его

**UC-03: Прохождение теста на внимание**

1. Пользователь открывает вкладку «Внимание»
2. В каждом из 5 заданий находит фигуру, отличающуюся от остальных
3. Система показывает итоговый балл и сохраняет результат

**UC-04: Просмотр результатов**

1. Пользователь открывает вкладку «Результаты»
2. Система загружает историю прохождений
3. Пользователь видит название теста, балл и дату

## Задача 7 — Техническое задание (ключевые разделы)

### Наименование системы
Веб-сервис когнитивных тренировок **Memory Trainer** (Тренажёр памяти)

### Цель создания
Создание веб-приложения для тренировки памяти и внимания у пожилых пользователей с сохранением истории результатов

### Требования к функциям системы (Реализованный функционал)

1. **Функция авторизации и регистрации** — безопасный stateless-доступ на основе JWT-токенов и BCrypt-хеширования паролей.
2. **Функция теста на память** — упражнение «Запоминание слов»: показ 5 слов, таймер 25 секунд, выбор из 10 вариантов, подсчёт балла.
3. **Функция теста на внимание** — упражнение «Найди лишнее»: 5 раундов, сетка 3×3, поиск отличающейся фигуры.
4. **Функция сохранения результатов** — запись баллов в PostgreSQL с привязкой к пользователю.
5. **Функция просмотра истории** — вкладка «Результаты» с датой, названием теста и счётом.

### Требования к надёжности
- Доступность сервиса при локальном развёртывании через Docker Compose
- Сохранность данных пользователей и результатов в PostgreSQL

## Задача 8 — Спецификация ПО (нефункциональные требования)

| Категория | Требование | Метрика |
|---|---|---|
| **Производительность** | Время ответа API | ≤ 1 секунды для операций авторизации и сохранения результата |
| **Надёжность** | Сохранность данных | Результаты не теряются при перезапуске контейнеров (volume PostgreSQL) |
| **Безопасность** | Хранение паролей | BCrypt-хеширование; JWT для защищённых эндпоинтов |
| **Совместимость** | Поддержка браузеров | Chrome, Firefox, Safari, Edge (последние 2 версии) |
| **Удобство использования** | Доступность для пожилых | Шрифт ≥ 20px, крупные кнопки, минимум отвлекающих элементов |
| **Развёртывание** | Docker Compose | Запуск frontend + backend + PostgreSQL одной командой |

## 🚀 Инструкция по локальному запуску

### Вариант А: Docker Compose (рекомендуется)

Из корня проекта:

```bash
docker compose up -d --build
```

| Сервис | URL |
|---|---|
| Frontend | http://localhost:3001 |
| Backend API | http://localhost:8080 |
| Health check | http://localhost:8080/api/health |

Остановка:

```bash
docker compose down
```

### Вариант Б: Локальная разработка

**1. PostgreSQL**

```bash
docker run --name memory-trainer-db \
  -p 5432:5432 \
  -e POSTGRES_DB=memory_trainer \
  -e POSTGRES_USER=memory_user \
  -e POSTGRES_PASSWORD=memory_pass \
  -d postgres:16-alpine
```

**2. Backend**

```bash
cd backend
mvn spring-boot:run
```

Параметры подключения к БД — в `backend/src/main/resources/application.yml`.

**3. Frontend**

```bash
cd frontend
npm install
npm run dev
```

Интерфейс: http://localhost:5173  
API по умолчанию: http://localhost:8080

**Типы тестов:**

## 👤 Сценарий использования для пожилого пользователя

1. Откройте сайт в браузере (рекомендуется увеличить масштаб страницы при необходимости).
2. Зарегистрируйтесь: придумайте простой логин и пароль (минимум 6 символов).
3. На вкладке **«Память»** или **«Внимание»** нажмите **«Начать тест»**.
4. Следуйте инструкциям на экране — не торопитесь.
5. После завершения результат автоматически сохранится.
6. Откройте вкладку **«Результаты»**, чтобы посмотреть историю.
